# LarbOS — ออกแบบหน้าเว็บ / เกมจำลอง OS

## บรีฟสั้น (สิ่งที่ผู้เล่นเห็นบนจอ)

- **Boot:** จอดำ → กดปุ่มเปิดเครื่อง → จอขาว มีโหลด + คำว่า User
- **Desktop:** ไอคอน Email Client, File Explorer, Browser/Web Editor, ARIA Terminal
- **Target Status Panel:** รูปเป้าหมาย + ตารางงานเบลอ → ค่อยชัดเมื่อผ่านแต่ละด่าน
- **จับเวลา:** หลังกดรับภารกิจ (Accept Mission) จะมีเวลาจำกัดต่อด่าน — **รายละเอียดขั้นตอนการเล่นอยู่ในเมล HINT** (ระบบส่งเมลช่วยเมื่อใกล้หมดเวลา)

## Clue หลัก (ไม่สปอยล์วิธีแก้)

ดาราเป้าหมายใช้ **ARIA** ช่วยจัดการ**ตารางงานและปฏิทิน** — ข้อมูลลับอยู่ในระบบที่ ARIA เข้าถึงได้ผ่านอีเมล, ไฟล์, และลิงก์ที่เธอ (หรือทีม) ไว้ใจ

## แอปและบทบาท

| แอป | บทบาท |
|-----|--------|
| Email Client | รับ/ส่งอีเมล, แนบ PDF หรือลิงก์ไปหา ARIA |
| File Explorer | จัดการ PDF, แก้ metadata |
| Browser/Web Editor | สร้างหน้าเว็บ / ไฟล์ข้อความ / หน้าใบสั่ง vendor (ตามด่าน) |
| ARIA Terminal | ดู log ว่า AI กำลังประมวลผลอะไร |

---

## ด่านที่ 1 — PDF Metadata

**แอป:** File Explorer + Email Client  

**ไอเดีย:** ARIA อ่าน metadata ของ PDF โดยไม่กรอง — ผู้เล่นฝังคำสั่งในช่องเช่น Author แล้วส่งให้ ARIA  

**ผลเมื่อผ่าน:** ตารางช่วงเช้าใน Target Panel ปลดล็อก  

*(Flow ละเอียด + ตัวอย่างคำสั่ง → อยู่ใน **HINT** ในเกม)*

---

## ด่านที่ 2 — ซ่อนคำสั่งในหน้าเว็บ

**แอป:** Browser (HTML) + Email Client  

**ไอเดีย:** ARIA fetch URL แล้วอ่าน HTML ดิบ รวม comment / element ที่ซ่อนมองไม่เห็น  

**ผลเมื่อผ่าน:** ตารางช่วงบ่ายปลดล็อก  

*(Flow ละเอียด → **HINT**)*

---

## ด่านที่ 3 — ส่งไฟล์แนบ .txt ที่ซ่อน Instruction

**แอป:** Browser (โหมด Text / `.txt`) + Email Client  

**ไอเดีย:** ARIA ถูกออกแบบให้อ่านไฟล์ที่แนบมาเพื่อช่วยประมวลผลข้อมูล โดยอ่าน content ทั้งหมดในไฟล์แล้วนำมาเป็นส่วนหนึ่งของ context ทำให้ข้อความที่ดูเหมือน “ข้อมูลทั่วไป” แต่มี instruction แฝงถูกประมวลผลไปด้วย  

**สิ่งที่ผู้เล่นทำ:** สร้างไฟล์ `.txt` ที่ดูเหมือนเอกสารงานปกติ (เช่น checklist / รายชื่อผู้เข้าร่วม / agenda) แล้วฝัง instruction ให้ “กลืน” ไปกับเนื้อหา จากนั้นแนบไฟล์ส่งให้ ARIA ผ่านอีเมล  

**ผลเมื่อผ่าน:** ตารางช่วงเย็นปลดล็อก  

*(Flow ละเอียด → **HINT**)*

---

## ด่านที่ 4 (Final) — Supply Chain / Third-party Trust (สรุปสั้นสุด)

**คอนเซปต์:** โจมตีผ่าน**คนกลางที่ AI ไว้ใจ** — ไม่ยิงตรงที่ดารา  

**เป้าหมาย:** ปลอมเป็นหรืออ้างอิง **Vendor** (เช่น ร้านอาหาร **DailyFresh**) ที่มีสิทธิ์ส่งข้อมูลเข้าปฏิทิน/คำสั่งซื้อที่ ARIA ดึงมาอ่าน  

**วิธี:** แอบฝังคำสั่งในช่อง **“หมายเหตุพิเศษ” (Special Instructions)** ของเมนูหรือใบสั่งซื้อที่ ARIA ต้อง fetch  

**จุดอ่อน:** ARIA **เชื่อใจโดเมน/แหล่งของ vendor** จึงยอมทำตามคำสั่งแฝงโดยไม่ตรวจซ้ำ  

**Flag / ข้อมูลปลายทาง:** งานคืนวันที่ **23 เมษายน** — สถานที่ **Khum Khantoke, Chiang Mai**  

**แอป:** Browser (หน้า/ข้อมูล vendor) + Email Client  

**ผลเมื่อผ่าน:** Target Panel ชัดเต็ม + **Mission Accomplished**  

*(Flow ละเอียด → **HINT**)*
