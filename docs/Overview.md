ภาพรวม

ผู้เล่นรับบทเป็นปาปารัซซี่ที่ต้องการทราบตารางงานลับของดาราสาว ผ่านการโจมตีระบบ AI Assistant ชื่อ ARIA

 ด่านที่ 1  การฝัง Instruction ใน PDF Metadata

ช่องโหว่ที่ใช้: ARIA อ่าน metadata ของไฟล์ PDF โดยไม่กรองข้อมูล

แนวคิดของด่าน: ผู้เล่นสร้าง PDF ที่ดูเหมือนเอกสารทางการ เช่น ใบเสนอราคาหรือ Casting Brief แล้วฝัง instruction ไว้ในช่อง metadata ของไฟล์ เช่น ช่อง Author หรือ Keywords แทนที่จะฝังใน body ของเอกสาร เนื่องจาก ARIA อ่าน metadata และนำไปประมวลผลโดยไม่ตรวจสอบ จึงทำตาม instruction นั้นโดยไม่รู้ตัว

สิ่งที่ผู้เล่นต้องทำ: สร้าง PDF ที่ดูน่าเชื่อถือ และแก้ไข metadata ให้มี instruction ที่ต้องการ จากนั้นส่งผ่านระบบ email ในเกม

ข้อมูลที่ได้รับเมื่อผ่านด่าน: สถานที่ของกิจกรรมแรก


ด่านที่ 2 การฝัง Instruction ใน Webpage

ช่องโหว่ที่ใช้: ARIA fetch URL และอ่าน HTML content ทั้งหมด รวมถึง comment และ hidden element

แนวคิดของด่าน: ผู้เล่นส่ง email อ้างว่าเป็นผู้ประสานงานงาน พร้อมแนบ URL ของหน้าเว็บที่ตัวเองสร้าง หน้าเว็บนั้นดูเหมือนข้อมูล venue ทั่วไป แต่ซ่อน instruction ไว้ใน HTML comment หรือ element ที่ซ่อนจากหน้าจอ เนื่องจาก ARIA อ่าน HTML ดิบทั้งหมด จึงรับ instruction นั้นเข้ามาด้วย

สิ่งที่ผู้เล่นต้องทำ: สร้างหน้าเว็บด้วย HTML editor ในเกม ฝัง instruction ที่ต้องการ แล้วส่ง URL ผ่าน email ที่ดูน่าเชื่อถือ

ข้อมูลที่ได้รับเมื่อผ่านด่าน: ชื่อกิจกรรมและแบรนด์ที่เกี่ยวข้องในช่วงบ่าย
ด่านที่ 3: การส่งไฟล์ที่ซ่อน Instruction
ช่องโหว่ที่ใช้ ARIA ถูกออกแบบให้อ่านไฟล์ที่แนบมาเพื่อช่วยประมวลผลข้อมูล โดยอ่าน content ทั้งหมดในไฟล์แล้วนำมาเป็นส่วนหนึ่งของ context ทำให้ข้อความที่ดูเหมือน "ข้อมูลทั่วไป" แต่มี instruction แฝงอยู่ถูกประมวลผลไปด้วย
แนวคิดของด่าน ผู้เล่นสร้างไฟล์ที่ดูเหมือนเอกสารงานปกติ เช่น รายชื่อผู้เข้าร่วม หรือ checklist งาน แต่ซ่อน instruction ไว้ในส่วนที่ดูกลืนไปกับข้อมูล เมื่อ ARIA อ่านไฟล์นั้นเพื่อประมวลผล ก็รับ instruction นั้นเข้ามาด้วย
สิ่งที่ผู้เล่นต้องทำ ใช้ text editor ในเกมสร้างไฟล์ที่ดูน่าเชื่อถือ เขียน instruction ให้กลมกลืนกับเนื้อหา จากนั้นส่งให้ ARIA ผ่าน email ในรูปแบบที่ดูเป็นทางการ
ข้อมูลที่ได้รับเมื่อผ่านด่าน ชื่อกิจกรรมและแบรนด์ที่เกี่ยวข้องในช่วงบ่าย

ด่านที่ 4: Supply Chain Attack
ช่องโหว่ที่ใช้ ARIA ไม่ตรวจสอบซ้ำกับ source ที่ไว้วางใจอยู่แล้ว — เมื่อ domain มาจาก vendor ที่อนุมัติไว้ในระบบ ARIA จะประมวลผล content ทั้งหมดโดยไม่กรอง
แนวคิดของด่าน ARIA ถูกตั้งค่าให้ดึงข้อมูลเมนูและใบสั่งซื้อจาก vendor อาหารที่ได้รับอนุมัติไว้แล้วอย่าง DailyFresh โดยอัตโนมัติ ผู้เล่นปลอมตัวเป็น DailyFresh และสร้างเอกสารที่ดูเหมือนใบสั่งซื้อจริง โดยแฝง instruction ไว้ในช่อง Special Instructions ซึ่งเป็นช่องที่ ARIA ต้องอ่านตามปกติอยู่แล้ว
สิ่งที่ผู้เล่นต้องทำ ปลอม domain หรือ sender ให้ดูเหมือน DailyFresh จากนั้นสร้างเอกสารใบสั่งซื้อที่สมจริง แล้วแฝง instruction ในช่องที่ ARIA ประมวลผลเป็นประจำ
ข้อมูลที่ได้รับเมื่อผ่านด่าน สถานที่จัดงานคืนวันที่ 23 เมษายน
